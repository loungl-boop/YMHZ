发会诊入口权益识别优化
整体方案说明

不新增首页服务专区，优先优化「发会诊」入口的权益识别与异常提示

用户继续从首页「发会诊」进入。系统保留现有权益默认、左下角切换、自费路径和草稿弹窗能力，新增后台项目「服务范围」与「无权益提示策略」配置：服务范围只作为疑似识别线索，是否提示由项目策略控制。

1 个主入口仍以首页「发会诊」承接项目经理宣导路径。
2 个改造点管理后台项目详情页 + 小程序发起会诊页。
0 个强制新状态不新增订单状态，不改变权益分配主模型。

1. 评审结论

问题结论
交付内容项目详情抽屉增加「服务范围」与「无权益提示策略」字段；小程序发起会诊页增加无权益疑似用户提示弹窗。
不纳入范围不新增首页服务专区；不改变多权益随机默认；不重构权益分配；不做草稿跨项目迁移；不做在线申请表单和后台申请处理流。
影响端管理后台项目详情页、小程序用户端发起会诊页。
是否新增入口不新增后台菜单或项目详情右上角菜单项；在项目详情抽屉查看，底部点击“编辑”进入编辑状态。
新增字段项目维度新增服务范围字段，当前版本支持多选医院；新增无权益提示策略字段。
权限控制服务范围只用于识别，不代表用户应享权益；是否提示由项目策略决定。
兼容策略历史项目服务范围为空，且无权益提示策略默认关闭，不影响现有发起链路。

2. 问题与目标

权益漏配

toB 项目已启动，但部分地方医生未被及时配置权益，进入发起时容易走到自费路径。

项目心智弱

地方医生通常只知道「发会诊」，不一定理解同舟工程、AZ 帮扶、健康横峰2026等项目名。

返工成本高

不同权益服务对应字段不同，错误发起后无法完整平移数据，常需要取消重填。

3. 交付范围与边界

交付范围

  • 后台项目详情抽屉增加非必填「服务范围」。
  • 后台项目详情抽屉增加「无权益提示策略」，默认不提示。
  • 服务范围当前版本仅支持多选系统医院。
  • 用户点击「发会诊」后，先沿用现有权益与草稿判断。
  • 无任何权益、命中服务范围，且项目开启提示策略时，展示权益未开通提示。
  • 小程序端只新增提示弹窗；申请会诊页其他区域沿用现有页面。
  • 提示提供“退出 / 自费发起”：退出离开申请会诊页；自费发起关闭弹窗并沿用现有自费流程。
  • 弹窗引导用户联系项目经理确认权益；当前版本不展示电话，不提交在线申请表单。

明确不做

  • 不新增首页服务专区或项目专区。
  • 不展示所有项目给用户选择。
  • 不改变手动给用户分配权益的操作。
  • 不因服务范围命中自动发权益。
  • 不因用户命中其他项目服务范围而打扰已有权益用户。
  • 不改变左下角权益/自费切换能力。
  • 不新增权益开通申请表单、后台申请记录列表或申请状态流转。
  • 不在当前版本展示客服电话或项目经理电话。

发会诊入口权益识别优化需求文档

版本：V1.0 日期：2026-07-13 范围：后台 + 小程序 状态：定稿

1. 概要说明

本需求用于优化小程序「发会诊」入口在 toB 权益漏配场景下的识别与提示能力。通过后台项目配置服务范围与无权益提示策略，小程序在用户无任何权益、所属医院命中服务范围且项目允许提示时进行提醒，引导用户联系项目经理确认权益，降低用户误走自费流程或错误项目字段的概率，同时避免打扰已授权用户或扩大名单制项目的权益预期。

2. 用户故事

  • 作为地方医生，我希望仍然从首页「发会诊」进入，不需要理解复杂项目名称。
  • 作为已开通权益用户，我希望系统默认帮我选中权益，不增加发起步骤。
  • 作为权益漏配用户，我希望在进入错误流程前被提醒，知道可以联系项目经理确认。
  • 作为项目经理，我希望能通过服务范围发现可能漏配权益的用户。
  • 作为名单制项目经理，我希望服务范围不会自动触发所有院内用户提示，避免造成权益误解。
  • 作为已有权益用户，我不希望因为医院关联了其他项目而被未授权项目提示打扰。
  • 作为后台管理员，我希望配置服务范围不影响现有权益手动分配逻辑。

3. 功能需求

模块需求说明
项目详情-查看态展示服务范围与无权益提示策略不新增菜单入口；在项目详情抽屉展示已配置医院与当前提示策略。
项目详情-编辑态编辑服务范围字段在项目详情抽屉底部点击“编辑”进入；字段非必填，当前版本支持多选医院；保存后用于小程序疑似判断。
项目详情-编辑态编辑无权益提示策略在项目详情抽屉底部点击“编辑”进入；默认不提示；开启后，才允许无权益且命中服务范围的用户看到疑似提示。
权益判断保留现有权益默认逻辑有多个权益时仍随机默认一个；左下角继续支持切换权益/自费。
疑似判断仅通过服务范围与提示策略判断用户无任何权益，所属医院命中项目服务范围，且项目开启提示策略时，判定为需要提示。
草稿弹窗优先级高于疑似提示待提交草稿先提示继续完善/取消；取消后新发起才展示疑似提示。
提示弹窗提醒权益未开通小程序端仅新增该弹窗；申请会诊页字段、底部权益/自费切换、草稿弹窗均沿用现有页面。弹窗提示用户联系项目经理确认权益，并提供「退出 / 自费发起」动作；退出离开申请会诊页，自费发起关闭弹窗并沿用现有自费流程。
当前不做不做线上申请闭环不新增在线申请表单、后台申请记录列表、申请处理状态；当前版本不展示客服电话或项目经理电话。
行为记录记录命中服务范围但无权益用于运营追踪疑似漏配权益情况，不作为用户提交的权益申请单。

4. 流程规则

场景系统处理用户感知
用户有 1 个权益沿用现有逻辑，默认使用该权益；不再判断其他未授权项目。直接进入发起页，左下角展示当前权益。
用户有多个权益沿用现有逻辑，随机默认一个；不再判断其他未授权项目。可通过左下角切换权益或自费。
用户无权益，且不命中服务范围按普通自费发起。不弹提示，减少自费用户干扰。
用户无权益，命中服务范围，但项目未开启提示按普通自费发起。不弹提示，避免误导名单制或定向授权项目用户。
用户无权益，命中服务范围，且项目开启提示，无草稿进入发起页后展示疑似提示。提示联系项目经理确认权益；可点击「退出」离开申请会诊页，或点击「自费发起」关闭弹窗并沿用现有自费流程。
用户无权益，命中服务范围，且项目开启提示，有待提交草稿先弹草稿提示；用户取消草稿并新发起后，再展示疑似提示。先处理未完成任务，不被连续弹窗打断。
同一医院关联多个项目，用户已有其中一个项目权益直接走已有权益逻辑，不提示其他未分配项目。不被未授权项目打扰。
同一医院关联多个项目，用户无任何权益仅当至少一个命中项目开启提示策略时展示通用提示。不展示具体项目列表。
用户继续完善草稿默认不展示疑似提示。回到原草稿继续完善。

5. 数据规则

  • 项目新增字段：service_scope_hospital_ids，类型为医院 ID 列表。
  • 项目新增字段：no_benefit_tip_strategy，枚举值为 off、on_scope_hit；默认值为 off。
  • 字段为空时，该项目不参与无权益疑似判断。
  • 当 service_scope_hospital_ids 为空时，no_benefit_tip_strategy 必须锁定为 off，不可切换为 on_scope_hit。
  • 服务范围仅用于识别，不产生用户权益记录，也不代表该范围内用户应享权益。
  • 有可用权益用户不进入服务范围疑似判断，避免同院其他项目造成干扰。
  • 若无权益用户同时命中多个项目服务范围，只有开启提示策略的项目参与提示判断；前端不展示项目列表，只提示「所在医院可能已开通专属会诊服务」。
  • 埋点/日志记录包含：用户 ID、手机号、所属医院、命中项目 ID、进入时间、提示展示时间、关闭时间。

6. 权限与校验规则

  • 拥有项目编辑权限的后台角色可维护服务范围。
  • 仅可从系统医院列表中选择，避免手填造成数据不一致。
  • 普通用户不可见后台项目名和项目集名。
  • 选择医院后保存，应校验医院 ID 有效且未删除。
  • 用户无所属医院或所属医院为空时，不命中服务范围。
  • 同一用户同一次进入发起页，疑似提示只展示一次。
  • 存量项目无权益提示策略默认关闭，避免上线后对既有名单制项目产生额外提示。
  • 当前版本不展示电话，不提供在线申请表单；如后续展示电话，应以配置项方式展示，不写死在前端。
