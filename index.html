会诊工单审核暂存与放行

本页用于向研发同学说明需求背景、产品方案思路和方案概览。默认展示整体方案，右上角可切换查看本次需求涉及的原型与 PRD。

核心结论：不新增主流程状态，仅在“待审核”节点内增加审核暂存状态：未预审 / 预审通过待放行。

1. 需求背景

当前会诊流程为：提交病历、病历审核、分诊、会诊、总结。部分 ToB 项目以会诊形式对地方医生进行培训，项目经理需要控制整体项目进度，同时医助人员数量有限，需要控制每天进入分诊和会诊的病例量。

由于不希望影响地方医生提交病历的积极性，平台不在提交环节限制申请数量。审核老师需要先完成病历质量判断，但对于已经审核无问题的病例，希望暂不大量流转到分诊。

2. 产品方案思路

不改变主流程

工单主状态仍按现有流程流转：待审核 → 待分诊 → 会诊 → 总结。避免影响状态机、统计、权限和历史数据。

审核与放行解耦

审核老师可选择“预审通过，暂不流转”，标记病历已看过且质量无问题，但不进入待分诊。

“预审通过待放行”不是主流程状态，而是待审核节点内的暂存分层。

3. 方案概览

页面改造点原则
待办页大病会诊 - 审核病历下增加“全部 / 未预审 / 预审通过待放行”；列表展示审核暂存状态。审核老师高频处理入口，强展示。
工单列表页不增加预审状态展示和筛选。全局列表保持简洁，避免字段膨胀。
工单详情页仅当待审核且预审通过待放行时展示预审信息，并提供取消预审操作。条件展示，支持就地修正。
工单审核页审核结果增加“预审通过，暂不流转”。合并在审核动作中，不做独立标记按钮。

4. 研发边界

事项结论
是否新增主流程状态不新增。主流程仍为待审核 → 待分诊。
是否影响驳回、不通过、通过不影响。三者继续沿用现有审核动作。
是否需要新字段需要。建议字段为 review_hold_status，枚举值为 none / preapproved_hold。

会诊工单审核暂存与放行需求文档

本页用于沉淀整体需求说明，包括目标、用户故事、功能需求、字段接口建议、测试验收和风险。

1. 需求概要

问题描述

ToB 培训型会诊项目中，地方医生提交病历数量较多，但医助、分诊、会诊资源有限。审核老师希望提前确认病历质量，但又不能让大量病例立即流转到待分诊。

解决方案

在不新增主流程状态的前提下，为待审核工单增加审核暂存状态：未预审 / 预审通过待放行。审核页新增“预审通过，暂不流转”。

2. 用户故事与验收标准

用户故事验收标准
作为审核老师，我希望将病历标记为预审通过但暂不流转，以便不用重复查看已确认无问题的病历。
  • 审核页增加“预审通过，暂不流转”。
  • 提交后主状态仍为待审核。
  • 审核暂存状态为“预审通过待放行”。
  • 不触发分诊流转。
作为审核老师，我希望在待办中区分未预审和预审待放行，以便优先处理真正未看过的病历。
  • 待办页“大病会诊 - 审核病历”下展示“全部 / 未预审 / 预审通过待放行”。
  • 待办列表展示“审核暂存状态”列。
  • 预审待放行工单操作列仍展示“审核”。
作为审核老师，我希望取消误标的预审通过状态，以便工单可以回到未预审池。
  • 待办页点击“预审通过待放行”状态标签可打开浮层。
  • 工单详情页在预审信息模块展示“取消预审”按钮。
  • 取消后审核暂存状态变为“未预审”。
非目标：不新增主流程状态；不限制地方医生提交病历数量；不改动现有“通过 / 驳回 / 不通过”的业务结果语义；工单列表页不新增审核暂存状态展示和筛选；本期不强制实现项目每日放行上限。

3. 功能需求

页面需求
待办页大病会诊 - 审核病历下增加“全部 / 未预审 / 预审通过待放行”筛选；列表增加“审核暂存状态”列；操作列保持“查看 / 审核 / 更多”。
工单列表页保持现状。不展示审核暂存状态，不增加审核暂存状态筛选。
工单详情页当工单状态为待审核且审核暂存状态为预审通过待放行时，顶部展示“预审通过待放行”标签，并展示预审老师、预审时间和取消预审按钮。
工单审核页审核结果调整为“通过并流转 / 预审通过，暂不流转 / 驳回 / 不通过”。提交不增加二次确认。

4. 技术规格

字段类型枚举/说明
review_hold_statusenumnone：未预审；preapproved_hold：预审通过待放行。
review_hold_operator_idstring最近一次设置预审通过待放行的操作人 ID。
review_hold_operator_namestring最近一次设置预审通过待放行的操作人姓名。
review_hold_atdatetime最近一次设置预审通过待放行的时间。
动作前置条件结果
设置预审通过待放行工单状态 = 待审核；用户有审核权限review_hold_status = preapproved_hold；写入操作人和时间。
取消预审工单状态 = 待审核；review_hold_status = preapproved_holdreview_hold_status = none；记录取消操作日志。
通过并流转工单状态 = 待审核；用户有审核权限复用原通过逻辑进入待分诊；不新增主流程状态。

5. 测试与验收

测试场景预期结果
待审核工单选择“预审通过，暂不流转”主状态仍为待审核；进入“预审通过待放行”筛选；不出现在待分诊。
预审待放行工单点击“审核”后选择“通过并流转”工单进入待分诊；原审核通过相关通知、任务、统计保持一致。
在待办页状态标签或详情页点击取消预审工单回到未预审列表；详情页不再展示预审通过待放行信息。
工单列表页查看待审核工单不展示审核暂存状态，不提供审核暂存状态筛选。
驳回、不通过、通过原流程回归所有原流程结果、通知、日志、状态流转保持原有行为。
