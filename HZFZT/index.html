会诊分诊台 PRD

版本：V1.0　范围：内部管理后台　目标：完成专家预约节点全流程线上管理。

1. 背景与目标

病历审核通过后，审核老师将病历通过微信发送给分诊老师，分诊老师与专家确认可会诊情况及时间，再由审核老师联系客户并录入最终结果。现状依赖线下转述，过程无统一状态、无清晰负责人、无专家邀约记录。新增会诊分诊台，将内部过程纳入系统，同时保持现有工单主流程及病历分享方式不变。

2. 功能范围

2.1 分诊工作台

  • 状态页签：待预约、待专家反馈、待客户确认、待重新分诊、已完成；列表不重复展示预约状态字段。
  • 节点人员快捷范围默认均为“全部”，顺序为全部/我的/公共池或未指定；已完成不显示人员快捷范围。
  • 公共筛选支持患者/工单搜索、会诊科室、项目、审核人、专家预约人、客户确认人。
  • 公共字段固定展示患者/工单、科室/项目、申请人/医院；阶段信息按状态命名为等待情况、专家邀约、待确认方案、退回原因/原方案、会诊安排。
  • 人员字段按当前阶段重排：当前节点负责人优先，已完成按审核人、专家预约人、客户确认人的流程顺序展示。
  • 申请人列展示脱敏手机号和查看入口；待客户确认阶段突出“拨打电话”，完整号码查看留痕，支持复制及调用系统拨号。

2.2 领取、指派与分享

  • 待预约任务默认进入公共池，可领取、指派、转派。
  • 病历链接随待预约任务直接可用，查看和复制链接与领取、选择专家完全解耦。
  • 选择专家支持多选；一次确认可为多位专家分别创建独立邀约记录。
  • 查看、展开或复制链接均不改变状态；点击“确认已分享给所选专家”后才记录分享人、分享时间并进入待专家反馈。

2.3 双视图与专家推荐

  • 列表视图用于检索、批量选择和批量操作；协同视图用于围绕单张工单连续完成当前节点操作。
  • 协同视图左侧为当前状态工单卡片，右侧按状态展示专家推荐、邀约维护、客户确认、重新分诊或最终结果。
  • 独立专家库入口暂不提供；待预约阶段在右侧根据当前工单动态推荐和搜索专家。
  • 支持综合推荐、项目会诊量、累计会诊量、当前分诊量排序，并展示医院、职称、擅长和业务数量。

2.4 专家反馈

  • 每次联系一位专家产生一条独立邀约记录，支持待反馈、可会诊、不可会诊、备选、客户确认中、已采用、未采用、已终止。
  • 待专家反馈阶段可新增分享专家、取消分享或更换专家；更换操作按“终止原邀约 + 新建邀约”实现，不覆盖历史。
  • 专家反馈时间以文本保存，允许录入多个可选时间或附加条件。
  • 支持对同一专家、相同反馈结果的多条任务批量记录或修改。

2.5 客户确认与重新分诊

  • 客户确认弹窗先选择处理结果：确认成功、仅记录沟通、客户不接受，再显示对应表单。
  • 仅确认成功时，最终专家、会诊日期和具体时间为必填；仅记录沟通不要求填写最终方案；客户不接受只要求原因和客户可接受条件。
  • 客户确认成功后，被选专家为已采用，其他仍有效专家为未采用；支持多条任务逐条录入后批量确认。
  • 重新分诊支持协调原专家、采用备选专家、邀请新专家三种完整表单；邀请新专家后进入待专家反馈。

2.6 跨节点指派

  • 所有未完成节点均支持批量指派专家预约人，并可提前指定客户确认人。
  • 跨节点指派只更新人员关系，不跳转业务状态、不自动生成专家反馈或客户确认结果。
  • 提前指定的客户确认人在任务进入待客户确认时自动承接。

2.7 二期医助排班预留

  • 会诊时间需同时满足专家、客户和医助容量，不以简单人员下拉框代替资源排班。
  • 护士长维护时段容量，客户确认具体时间时先占用一个医助名额，预约取消或改期时自动释放。
  • 具体医助可在预约完成后由护士长分配；一期的最终会诊安排数据结构预留医助资源关联。

3. 权限与数据

能力控制说明
查看分诊信息叠加现有工单数据范围；所有有权查看工单的人员均可查看预约进度。
专家预约操作控制领取、选择专家、分享、记录专家反馈、重新分诊。
客户确认操作控制客户沟通记录、确认成功、退回重新分诊。
分配管理控制指派和转派专家预约人、客户确认人。
异常处理控制终止预约、恢复、修正最终结果等管理动作。

4. 企业微信通知

仅在指派/转派专家预约人、提交客户确认、变更客户确认人、退回待重新分诊时发送。发送失败不回滚业务操作，保留失败状态并支持重试。客户确认成功、最终专家或时间修改、工单取消、预约终止均不发送。

5. 验收标准

  1. 病历审核通过后自动出现待预约任务，不改变原工单主流程。
  2. 五个状态均可在列表视图和协同视图之间切换，筛选与业务状态保持一致。
  3. 协同视图选择左侧工单时，只刷新右侧工作区，不产生领取或状态变化。
  4. 待预约右侧支持多选专家和分享；其他四个状态提供与当前节点匹配的连续操作。
  5. 多位专家邀约彼此独立，反馈文本、状态、操作者和时间完整留痕。
  6. 客户确认成功后最终专家和具体时间正确回写，其他有效专家变为未采用。
  7. 所有操作同时通过数据权限、操作权限和状态条件校验。
